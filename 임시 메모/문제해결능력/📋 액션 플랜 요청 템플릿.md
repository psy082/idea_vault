# 맥락: 문제 해결 능력 개발에 대한 연구 기반 이해

## 핵심 원칙 (연구 기반)
1. **문제 표상이 해결의 핵심**: 문제를 어떻게 이해하고 프레이밍하느냐가 해결 방법을 결정함 (Chi et al., 1981; Newell & Simon, 1972)
2. **전문가와 초보자의 차이**: 전문가는 심층 원리로, 초보자는 표면 특징으로 문제를 범주화함
3. **훈련 가능성**: 체계적 훈련으로 7-12주 내에 측정 가능한 향상 확인됨 (RCT 연구 다수)
4. **영역 특수성**: 문제 해결 능력은 특정 영역에서의 대량 연습이 필요함 (최소 10년/10,000시간)

## 효과적인 훈련 구조 (Ericsson, 2008; Sweller, 1988)
- **1단계**: Worked examples 학습 (해결된 문제 분석)
- **2단계**: 점진적 난이도 증가 (쉬운 → 어려운)
- **3단계**: 즉각적 피드백 + 반복 수정
- **4단계**: 메타인지 전략 (계획-모니터링-평가)
- **일일 권장**: 1-4시간 집중 연습 (그 이상은 효과 감소)

## 내 상황
- **영역**: [여기에 구체적 영역 입력 - 예: "프론트엔드 성능 최적화", "React 아키텍처 설계", "시스템 디버깅"]
- **현재 수준**: [초보자/중급/고급]
- **가용 시간**: [하루 X시간, 주 Y일]
- **기간**: [X주/X개월]
- **구체적 목표**: [예: "대규모 좌표 렌더링 성능 문제를 독립적으로 진단하고 해결할 수 있기"]

## 요청사항
위 연구 기반 원칙을 적용하여, 내 영역과 상황에 맞는 **12주 점진적 훈련 플랜**을 작성해주세요:

1. **주차별 학습 목표** (구체적 문제 유형)
2. **Worked examples 리스트** (각 주차에 분석할 예시 문제)
3. **실습 문제** (난이도 순서대로)
4. **메타인지 체크리스트** (각 문제 해결 후 자문할 질문들)
5. **피드백 방법** (스스로 또는 외부에서 피드백 받는 방법)
6. **진도 측정 지표** (향상도를 어떻게 측정할지)

특히 강조해주세요:
- 문제를 "어떻게 볼 것인가"에 대한 프레이밍 연습
- 표면 특징이 아닌 심층 원리로 문제를 범주화하는 훈련
- 인지 부하를 고려한 단계적 접근
