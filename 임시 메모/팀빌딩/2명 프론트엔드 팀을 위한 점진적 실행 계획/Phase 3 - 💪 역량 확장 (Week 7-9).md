### 핵심 목표

**파트원 전문 영역 확보 + 역 멘토링 시작**

### Week 7: 전문성 재분배

#### 파트원 전문 영역 선택

```
파트원이 "담당 전문가"가 될 영역 선택:

옵션 (관심사와 강점 기반):
1. 접근성(Accessibility) 전문가
2. 애니메이션/인터랙션 전문가
3. 테스팅 전략 전문가
4. 성능 모니터링 전문가

선택 기준:
- 파트원의 흥미
- 팀의 필요성
- 파트리더가 깊이 다루지 않는 영역

이번 주 목표: 
해당 영역의 팀 "go-to person" 되기
```

#### 역 멘토링 시작

```
파트원 → 파트 리더 지식 전달:

예시 (접근성 선택 시):
1. 파트원이 1주일간 학습
2. 금요일: 30분 세션에서 파트리더에게 설명
3. 실제 코드베이스에 적용 계획 수립

효과:
- 파트원: 설명하며 더 깊이 이해
- 파트리더: 새로운 관점 획득
- 팀: 전문성 분산
```

### Week 8: 고급 협업 패턴

#### 비동기 협업 강화

```
페어 프로그래밍 조정: 주 2회 → 주 1회

대신 강화:
1. 비동기 코드 리뷰
   - 리뷰어가 "교육적 코멘트" 작성
   - 예: "이렇게 하면 왜 더 좋은지 3가지 이유"
   
2. 비디오 설명
   - 복잡한 PR: Loom으로 5분 설명 영상
   - 파트원이 이해 못한 부분 타임스탬프로 질문
   
3. 문서 우선 접근
   - 새로운 패턴 도입 시 짧은 문서 먼저 작성
   - 팀이 리뷰 후 적용
```

### Week 9: 위기 대응 연습

#### 복잡한 버그를 함께 해결

```
"교육용 버그 헌팅" 세션 (2시간):

1. 복잡한 실제 버그 선택
2. 파트원이 먼저 30분 디버깅 시도
3. 파트리더와 함께 나머지 해결
4. 회고:
   - 어떤 접근법이 효과적이었나?
   - 어떤 지식이 부족했나?
   - 다음번엔 어떻게 할까?
```
