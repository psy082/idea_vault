### 잘못된 방향의 징후

```
❌ 파트원이 질문을 안 한다
   → 심리적 안전성 문제
   → 해결: 파트리더가 먼저 실수 공유하기

❌ 페어 프로그래밍이 일방적 강의가 된다
   → 파트원 참여 부족
   → 해결: Navigator 역할 강화, 더 많은 질문 유도

❌ 코드 리뷰가 지나치게 길어진다
   → 기대치 불일치
   → 해결: 코드 리뷰 가이드 재정립

❌ 파트원의 독립 작업 시간이 과도하게 길다
   → 막혀있지만 질문 안 하는 것
   → 해결: "30분 rule" - 30분 막히면 무조건 질문

❌ 팀 속도가 급격히 감소
   → 멘토링 오버헤드가 너무 큼
   → 해결: 페어 프로그래밍 빈도 조정
```

### 조정이 필요한 순간

```
상황별 대응:

프로젝트 데드라인 급박:
- 멘토링 일시 중단 (명시적 합의)
- 각자 독립 작업으로 전환
- 데드라인 후 회고로 배운 점 정리

파트원이 너무 빠르게 성장:
- Phase를 더 빨리 진행
- 더 높은 자율성 부여
- 새로운 도전 과제 제공

파트원이 어려워하는 경우:
- Phase를 천천히 진행
- 페어 프로그래밍 비중 높이기
- 더 작은 단위로 쪼개기
```
