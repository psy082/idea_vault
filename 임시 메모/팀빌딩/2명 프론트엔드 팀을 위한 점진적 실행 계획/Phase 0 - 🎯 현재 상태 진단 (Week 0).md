### 목표

팀의 현재 상태를 객관적으로 파악하고 개선 우선순위 설정

### 실행 내용

#### 1. 팀 효과성 자가 진단 (60분)

두 사람이 각각 아래 항목을 1-5점으로 평가 후 비교

**심리적 안전성** (가장 중요)

- [ ] 실수를 해도 비난받지 않는다고 느낀다
- [ ] 어려운 문제와 이슈를 자유롭게 제기할 수 있다
- [ ] 모르는 것을 물어봐도 괜찮다고 느낀다

**역량 격차 인식**

- [ ] 서로의 강점과 약점을 명확히 알고 있다
- [ ] 배우고 싶은 것을 공유하고 있다
- [ ] 도움이 필요할 때 요청하기 편하다

**목표 명확성**

- [ ] 우리 팀의 목표가 무엇인지 명확하다
- [ ] 각자의 역할과 책임이 명확하다
- [ ] 성공의 기준이 무엇인지 안다

**현재 협업 방식**

- [ ] 코드 리뷰가 효과적으로 이루어진다
- [ ] 지식 공유가 자연스럽게 일어난다
- [ ] 의사소통이 원활하다

#### 3. 학습 목표 설정 (30분)

**파트원 목표 (3개월)**

1. React 상태 관리 패턴 이해하고 적용하기
2. 복잡한 컴포넌트 구조 설계할 수 있기
3. 코드 리뷰에서 적극적으로 질문하고 배우기

**파트리더 목표 (3개월)**

1. 효과적인 멘토링 스킬 개발하기
2. 파트원이 독립적으로 작업할 수 있도록 지원하기
3. 팀 문서화와 지식 공유 체계 구축하기
