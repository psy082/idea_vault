### 핵심 목표

**심리적 안전성 구축 + 기본 협업 패턴 확립**

### Week 1: 관계와 신뢰 구축

#### 월요일: 팀 킥오프 미팅 (90분)

```
1. 서로 알아가기 (30분)
   - 개발자로서의 여정
   - 일하는 스타일 (선호하는 시간대, 커뮤니케이션 방식)
   - 기대와 우려사항

2. 협업 규칙 정하기 (30분)
   - 코드 리뷰 응답 시간: 24시간 이내
   - 질문하는 방법: Slack에 컨텍스트와 함께
   - 페어 프로그래밍 시간: 주 2회, 각 2시간
   - 회고 시간: 금요일 오후 4시

3. 첫 3주 목표 설정 (30분)
   - 측정 가능한 구체적 목표
```

#### 화-목: 페어 프로그래밍 집중 (주 2회, 각 2시간)

```
세션 구조:
1. 준비 (15분)
   - 오늘 다룰 작업 선택 (중간 난이도)
   - 학습 목표 명시

2. 페어 작업 (90분)
   - Driver/Navigator 역할 15분마다 교체
   - Navigator는 "왜 그렇게 하나요?" 질문 적극 장려
   - 막히면 멈추고 함께 토론

3. 회고 (15분)
   - 무엇을 배웠나?
   - 다음번에 개선할 점은?
```

#### 금요일: 주간 회고 (30분)

```
체크포인트:
✅ 심리적 안전성
   - 질문하기 편했나?
   - 실수를 편하게 공유했나?

✅ 학습 성과
   - 이번 주 배운 것 3가지
   - 아직 모호한 부분

✅ 다음 주 조정사항
```

### Week 2: 지식 공유 체계 구축

#### 새로운 추가 요소

**1. TMS(교류기억체계) 시작**

```
월요일 체크인 (15분):
- 이번 주 각자 작업할 영역
- 서로에게 도움 요청할 가능성이 있는 부분
- "이건 내가 잘 아니까 물어봐" 명시

예시:
파트리더: "이번 주 상태 관리 리팩토링 할 건데, 
        Zustand 관련해서는 언제든 물어봐"
파트원: "신규 기능 UI 구현할 건데, 
        CSS 애니메이션은 내가 잘 아니까 필요하면 도와줄게"
```

**2. 학습 문서 시작**

```
공유 노트북 (Notion/Obsidian) 구조:
/팀 학습 일지
  /Week 2
    - 배운 개념: React Query 캐싱 전략
    - 해결한 문제: 무한 스크롤 성능 이슈
    - 다음에 알아볼 것: Suspense 패턴
    
파트원이 작성 → 파트리더가 리뷰/보완
```

### Week 3: 독립성 향상

#### 점진적 자율성 모델 적용

```
작업 배분 전략:

[파트리더 전담]
- 아키텍처 결정
- 복잡한 성능 최적화
- 코드 리뷰 최종 승인

[파트원 전담 - 파트리더 서포트]
- 잘 정의된 신규 기능 (컴포넌트 3개 이하)
- UI 구현 (디자인 시스템 기반)
- 단순 버그 수정

[함께]
- 복잡한 신규 기능 (페어 프로그래밍)
- 리팩토링
- 기술 부채 해결
```

### Phase 1 측정 지표

**과업 성과**

- [ ] 배포 횟수: 주 2회 이상 유지
- [ ] 버그율: 증가하지 않음
- [ ] 코드 리뷰 시간: 평균 4시간 이내

**팀원 태도**

- [ ] 주간 만족도 설문: 3.5/5 이상
- [ ] "질문하기 편했다": 4/5 이상

**팀 역량**

- [ ] 파트원: 새로운 개념 3개 학습
- [ ] 파트리더: 멘토링 개선점 3개 식별
